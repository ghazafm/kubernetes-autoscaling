# Perbandingan Reward Function (Stable Simple)

Dokumen ini membandingkan dua fungsi di `agent/stable_simple/environment/environment.py`:

- Fungsi A (existing): `calculate_reward` (`agent/stable_simple/environment/environment.py:15`)
- Fungsi B (new): `new_calculate_reward` (`agent/stable_simple/environment/environment.py:41`)

## 1) Rumus Kedua Fungsi

### Fungsi A (`calculate_reward`)

```python
response_time_penalty = response_time / 100.0
cost = action / 99.0
cost_weight = max(0.0, 1.0 - response_time_penalty)
cost_penalty = cost * cost_weight
reward = 1.0 - (response_time_penalty + cost_penalty)
```

Bentuk ringkas:

`reward_a = 1 - r - c * max(0, 1 - r)`

### Fungsi B (`new_calculate_reward`)

```python
rt_penalty = response_time / 100.0
cost = action / 99.0
reward = (1.0 - rt_penalty) * (1.0 - cost)
```

Bentuk ringkas:

`reward_b = (1 - r)(1 - c) = 1 - r - c + rc`

Dengan:

- `r = response_time / 100`
- `c = action / 99`

## 2) Bentuk Piecewise dan Relasi Matematis

### Fungsi A (piecewise)

- Jika `r <= 1` (`response_time <= 100`):
  `reward_a = 1 - r - c(1 - r)`
- Jika `r > 1` (`response_time > 100`):
  `reward_a = 1 - r`

### Fungsi B

- Selalu:
  `reward_b = (1 - r)(1 - c)`

### Relasi A vs B

- Untuk `r <= 1`: `reward_a = reward_b` (identik)
- Untuk `r > 1`:
  - `reward_a - reward_b = (1 - r)c <= 0`
  - Artinya `reward_a <= reward_b`
  - Sama hanya saat `c = 0` (action = 0)

## 3) Kondisi Reward Ekstrem

### Fungsi A

- Reward maksimum: `1.0000` pada `action=0`, `response_time=0`
- Reward minimum:
  - Teoritis: tidak berbatas bawah saat `response_time -> inf`
  - Jika dibatasi `response_time <= 300`: minimum `-2.0000`
- Reward `= 0`:
  - `response_time = 100` untuk action apa pun
  - atau `action = 99` dan `response_time <= 100`

### Fungsi B

- Reward maksimum: `1.0000` pada `action=0`, `response_time=0`
- Reward minimum:
  - Teoritis: tidak berbatas bawah saat `response_time -> inf` dan `action < 99`
  - Jika dibatasi `response_time <= 300`: minimum `-2.0000` (saat `action=0`, `response_time=300`)
- Reward `= 0`:
  - `response_time = 100` (action apa pun)
  - atau `action = 99` (response time apa pun)

## 6) Implikasi Praktis

- Pada `response_time <= 100`, kedua fungsi memberi reward yang sama persis.
- Pada `response_time > 100`, fungsi baru (B) memberi reward lebih tinggi (kurang negatif), terutama untuk action besar.
- Efek paling kontras: `action=99` dengan `response_time > 100`:
  - Fungsi A tetap negatif (`1 - r`)
  - Fungsi B selalu `0`
