# Perbandingan Reward Function

Dokumen ini membandingkan:

- Fungsi A (existing): `environment/environment.py:15`
- Fungsi B (alternatif linear):

```python
response_time_penalty = response_time / 100.0
cost_penalty_raw = action / 99.0
cost_weight_multiplier = max(0.0, 1.0 - response_time_penalty)
effective_cost_penalty = cost_penalty_raw * cost_weight_multiplier
total_penalty = response_time_penalty + effective_cost_penalty
reward = 1.0 - total_penalty
```

Asumsi perhitungan perbandingan:

- Fungsi A: `w_rt = 1.0`, `w_cost = 1.0`
- Pembulatan 4 desimal

## 1) Kondisi Reward Maksimum, Minimum, dan Nol

### Fungsi A (existing, piecewise)

- Reward paling besar:
  - `reward = 1.0000`
  - Kondisi: `response_time <= 50` dan `action = 0`
- Reward paling kecil:
  - Teoritis: tidak berbatas bawah (`-> -inf`) saat `response_time -> inf`
  - Dalam tabel ini (`response_time <= 300`): minimum `-2.7500` pada `response_time = 300` (action apa pun)
- Reward `= 0`:
  - Jika `action = 99` dan `response_time <= 80`
  - Atau `response_time = 80` (dengan action apa pun)

### Fungsi B (alternatif linear, `reward = 1.0 - total_penalty`)

- Reward paling besar:
  - `reward = 1.0000`
  - Kondisi: `response_time = 0` dan `action = 0`
- Reward paling kecil:
  - Teoritis: tidak berbatas bawah (`-> -inf`) saat `response_time -> inf`
  - Dalam tabel ini (`response_time <= 300`): minimum `-2.0000` pada `response_time = 300` (action apa pun)
- Reward `= 0`:
  - Untuk `0 <= response_time < 100`: saat `action=99`
  - Untuk `response_time = 100`: action apa pun

## 4) Catatan Cepat

- Nilai `selisih_a_minus_b` positif berarti reward Fungsi A lebih tinggi dari Fungsi B.
- Nilai `selisih_a_minus_b` negatif berarti reward Fungsi B lebih tinggi dari Fungsi A.
- Pada `response_time=0`, reward kedua fungsi identik untuk semua action.
